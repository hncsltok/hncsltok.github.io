Escaped Character

Inside a string, some character " can cause ambiguity. To solve this problem, escape character \ is used.

#include <litan.h>

int main() {

    // this statement is correct, as `"` in string is escaped using `\`
    put_string("I want put \" inside a string !");

    return 0;
}
#include <litan.h>

int main() {

    // this statement has ambiguity and will cause an error
    put_string("I want put " inside a string !");

    return 0;
}

The following table is about how to escape an character in string.

Original Character Escaped Character
" \"
' \'
\ \\
<enter> \n
<tab> \t
<terminal character> \0

Like <enter> above, a term enclosed by < and > braces is used in this tutorial to refer some grammatical concept, or something unprintable.

If we say something like <expression>; is an expression statement, <expression>; itself is not an expression statement. But something such as put_string("Hello"); is an expression statement, as put_string("Hello") is an <expression>.
